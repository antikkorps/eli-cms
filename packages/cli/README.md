# @eli-cms/cli

CLI for Eli CMS. Generates TypeScript interfaces from your content types and components â€” zero manual typing.

## Setup

### 1. Build

```bash
pnpm build:cli
```

### 2. Configure (optional)

The CLI needs to know your API URL. By default it uses `http://localhost:8080`.

You can override it with:

- `--url` flag (highest priority)
- `ELI_API_URL` environment variable
- `.elicms.toml` in the current directory or home directory

```toml
# .elicms.toml
[api]
url = "http://localhost:8080"
```

> **Priority order:** `--url` flag > `ELI_API_URL` env var > `./.elicms.toml` > `~/.elicms.toml` > `http://localhost:8080`

## Commands

### `eli codegen`

Fetches all content types and components from the public `/api/v1/public/schema` endpoint and generates a `eli-cms.d.ts` file with typed interfaces.

```bash
# Generate eli-cms.d.ts in the current directory
eli codegen

# Specify API URL
eli codegen --url http://my-cms.example.com

# Custom output path
eli codegen --output ./types/cms.d.ts

# Both
eli codegen --url http://my-cms.example.com --output ./src/types/eli-cms.d.ts
```

> No authentication required â€” the schema endpoint is public.

## Generated Output

Given a CMS with a "Blog Post" content type and a "Hero" component, the generated file looks like:

```typescript
// Auto-generated by @eli-cms/cli â€” do not edit
// Generated from: http://localhost:8080

/** Component blocks */
export interface HeroBlock {
  _component: 'hero';
  heading: string;
  subheading?: string;
}

/** Content types */
export interface BlogPost {
  title: string;
  body?: string;
  coverImage?: string;
  category?: 'tech' | 'news';
  gallery?: BlogPostGalleryItem[];
}

export interface BlogPostGalleryItem {
  image?: string;
  caption?: string;
}

/** Type maps */
export interface EliContentMap {
  'blog-post': BlogPost;
}

export interface EliComponentMap {
  hero: HeroBlock;
}
```

### Type Mapping

| Field Type                                     | TypeScript Type            |
| ---------------------------------------------- | -------------------------- |
| `text`, `textarea`, `email`, `url`, `richtext` | `string`                   |
| `number`                                       | `number`                   |
| `boolean`                                      | `boolean`                  |
| `date`                                         | `string` (ISO 8601)        |
| `author`                                       | `string` (user ID)         |
| `select` (with options)                        | `'opt1' \| 'opt2'`         |
| `select` (no options)                          | `string`                   |
| `media`                                        | `string` (UUID)            |
| `media` + `multiple`                           | `string[]`                 |
| `repeatable`                                   | `{Parent}{Field}Item[]`    |
| `component`                                    | `({Block1} \| {Block2})[]` |

### Using the Generated Types

```typescript
import type { EliContentMap, BlogPost } from './eli-cms';

// Type-safe content fetching
async function getContent<T extends keyof EliContentMap>(slug: T): Promise<EliContentMap[T][]> {
  const res = await fetch(`/api/v1/public/contents/by-type/${slug}`);
  const json = await res.json();
  return json.data;
}

const posts = await getContent('blog-post'); // BlogPost[]
```
